# Code Ownership

Who owns which part of the MindDrop codebase. Aligned with [MASTER_ARCHITECTURE.md §2–6](./MASTER_ARCHITECTURE.md) — this file is the granular per-path reference.

**Owner keys:**
- **Lovable** — edited via the Lovable agent (web team).
- **Android Studio** — Kotlin engineers only, never Lovable.
- **Supabase** — schema/data managed via migrations from Lovable.
- **GitHub** — CI/CD pipelines and repo config.
- **Developer** — humans, either web or Android.
- **Never edit manually** — auto-generated by tooling.
- **Future ownership** — folder does not exist yet; owner assigned in advance.

## Ownership Rules

1. **One owner per file.** If two roles could edit the same file, the file itself declares a header comment naming the owner.
2. **Auto-generated files carry a `DO NOT EDIT` header** and are excluded from human review.
3. **Kotlin never touches `src/`.** Web never touches `android/`. This is the merge-conflict-free boundary.
4. **Migrations are append-only.** Never edit an existing file in `supabase/migrations/`.
5. **Docs (`docs/`) are shared.** PR-only edits from Android Studio; Lovable can edit directly.
